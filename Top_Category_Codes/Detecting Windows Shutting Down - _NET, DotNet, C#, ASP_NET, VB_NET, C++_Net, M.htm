Detecting Windows Shutting Down

Print
Email
  
Submitted on: 3/12/2004 11:45:49 AM
By: Wilbert Peart 
Level: Beginner
User Rating: By 2 Users
Compatibility:C#

Users have accessed this article 3512 times.
 

(About the author)
 
     Usually you build applications assuming your users will click on the Exit button, or whatever procedure you decide to finish your application properly, but what happens when the user, just shutdown his/her computer, maybe you would like to save some info in a log file or so... This code will let you know when a user is shutting down the computer, so you can perform some action before it happens. Note that this code should be placed in the main form. For this example the main form is called frmLogin.

/*This is the Message constant Windows sent to let the application to let it know is going to shutdown*/
private const int WM_QUERYENDSESSION=0x0011;
private bool isShuttingDown=false;
/*This is the method (WndProc) which receive Windows Messages we are overriding it to make it works as we want*/
protected override void WndProc(ref Message m)
{
     if(m.Msg==WM_QUERYENDSESSION)
     {
          isShuttingDown=true;
     }
base.WndProc(ref m);
}
private void frmLogin_Closing(object sender, System.ComponentModel.CancelEventArgs e)
{
     if(isShuttingDown)
          {
               if(MessageBox.Show(this,"The application is still running, are you sure you want to exit?",
                    "Confirm Closing by Windows Shutdown",MessageBoxButtons.YesNo,MessageBoxIcon.Warning)==System.Windows.Forms.DialogResult.Yes)
               {
                    e.Cancel=false;
               }
               else
                    e.Cancel=true;
          }
}
