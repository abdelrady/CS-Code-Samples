The call to Marshal.AllocHGlobal is wasteful as you'll turn right around and overwrite the value.  Remove that call.  Be aware that unless somebody cleans up the VirtualAlloc call you'll have a memory leak.  Remove the call to new for the structure as it is redundant.  As for your structure you can remove the unsafe modifier as the code is safe.  Same thing for the extern call.

 

Apply StructLayout(LayoutKind.Sequential) to the structure to enforce the layout.  That is the default for structs but just to be safe.  Other than that the code looks reasonable.  The only thing that is remotely raising a flag is the bool in the structure but it should really be fine.  I guess I'm thinking about padding on the unmanaged code side here.  If you look in memory at the address pointed to by pnt can you confirm that the correct values were set from the unmanaged code?  If the memory is wrong then your structure will be wrong.

Answer Re: c++ pointer structure to C#
Was this post helpful ?

You need to attribute your bool in the struct.  The default marshalling for a bool is 4 bytes.

http://msdn2.microsoft.com/en-us/library/t2t3725f.aspx

 

[StructLayout(LayoutKind.Sequential)]

public struct PTImgInfo

{

[MarshalAs(UnmanagedType.U1)]

public bool bColor;

public UInt16 wWidth;

public UInt16 wHeight;

}
