wTo find the smallest element construct a tournament as follows: Compare all the numbers in pairs.
wOnly the smallest number of each pair is potentially the smallest of all so the problem is reduced to size n/2 . Continuing in this fashion until there is only one left clearly solves the problem.
wExactly n - 1 comparisons are needed since the tournament can be drawn as an n-leaf binary tree which has n - 1 internal nodes (show by induction on n). Each of these nodes correspond to a comparison.
wWe can use this binary tree to also locate the second smallest number. The path from the root to the smallest element (of height lg n) must contain the second smallest element. Conducting a tournament among these uses lg n - 1 comparisons.
wThe total number of comparisons are: n - 1 + lg n - 1 = n + lg n - 2.
