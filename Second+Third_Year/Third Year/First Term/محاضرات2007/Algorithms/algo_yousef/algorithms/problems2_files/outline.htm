9.1.3
  • Since the depth of a decision tree is lg n!, it takes at least that many comparisons.
  • lg n!/2 = lg n! – lg 2 = lg n! – 1 = Q(n lg n)
  • Therefore, there is no comparison sort whose running time is linear on at least ½ of n! inputs of length n.
  • lg n!/2n = lg n! – lg 2n = lg n! – n = Q(n lg n) – Q(n) = Q(n lg n)
  • Therefore, there is no comparison sort whose running time is linear on at least 1/2n of n! inputs of length n.

9.1.1
  • To find the smallest element construct a tournament as follows: Compare all the numbers in pairs.
  • Only the smallest number of each pair is potentially the smallest of all so the problem is reduced to size n/2 . Continuing in this fashion until there is only one left clearly solves the problem.
  • Exactly n - 1 comparisons are needed since the tournament can be drawn as an n-leaf binary tree which has n - 1 internal nodes (show by induction on n). Each of these nodes correspond to a comparison.
  • We can use this binary tree to also locate the second smallest number. The path from the root to the smallest element (of height lg n) must contain the second smallest element. Conducting a tournament among these uses lg n - 1 comparisons.
  • The total number of comparisons are: n - 1 + lg n - 1 = n + lg n - 2.

12.2.5
  • Show that if a node in a binary search tree has two children then its successor has no left child and its predecessor has no right child.
  • Let v be a node with two children. The nodes that immidiately precede v must be in the left subtree and the nodes that immidiately follow v must be in the right subtree. Thus the successor s must be in the right subtree and s will be the next node from v in an inorder walk. Therefore s cannot have a left child since this child since this would come before s in the inorder walk. Similarly, the predecessor has no right child.

22.1.3
  • The transpose of a directed graph GT can be computed as follows: If the graph is represented by an adjacency-matrix A simply compute the transpose of the matrix AT in time O(V2). If the graph is represented by an adjacency-list a single scan through this list is sufcient to construct the transpose. The time used is O(E + V).
  • The square of a graph can be computed as follows: If the graph is represented as an adjacency matrix: A simply compute the product A2 where multiplication and addition is exchanged by anding and oring. Using the trivial algorithm yields a running time of O(n3). Using Strassens algorithm improves the bound to O(nlg 7) = O(n2.81).
  • If we are using the adjacency-list representation we can simply append lists eliminating duplicates. Assuming the lists are sorted we can proceed as follows: For each node v in some list replace the v with Adj[v] and merge this into the list. Each list can be at most V long and therefore each merge operation takes at most O(V) time. Thus the total running time is O((E+V)V) = O(E+V2).
