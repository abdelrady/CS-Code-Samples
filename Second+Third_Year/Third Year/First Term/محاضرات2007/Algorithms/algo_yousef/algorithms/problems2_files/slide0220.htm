wSince the minimum element in a tree is then left-most node, the worst-case is if the tree is built by inserting elements in decreasing order so that it is a left-going list.  In this situation TREE-MINIMUM must traverse the entire tree, which happens in Q(n)-time.  However, in the average (and best) case the height of the tree is Q(lg n), so TREE-MINIMUM will run in Q(lg n)-time.
wTREESUCCESSOR processes each subtree before proceeding to the next subtree outside the current one.  Therefore, it has to traverse each edge no more than twice.
w2n = O(n)
wO(n) + O(n) = O(n)
