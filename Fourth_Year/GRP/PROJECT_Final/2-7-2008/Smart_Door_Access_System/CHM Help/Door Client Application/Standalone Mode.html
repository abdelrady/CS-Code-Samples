Current & Dafult Configuration : Standalone

This mode can be manually or the client enter through  it automatic after the network  unavailability event detected.
To configure Door to run in standalone mode from the "Door Mode" drop down menu Select "Standalone"also
set "Text Log" to "On" to log entrance attempts in the file name specified in "Text Log File Name ".
Don’t forget to add the barcodes that is allowed to enter door in the file name specified in "Standalone Mode Students File" as shown .

Figure (7) The Manual Entry of Allowed barcodes numbers

Standalone mode when  it used as a Temporary mode in case of network unavailability to do so you have to:

  1. Set "Door Mode" To "Connect to Doors Server".
  2. Check The"Auto Connect Label".
  3. Specify the time out between each attempt to connect in the "Timeout" Text Field and.
  4. Specify the number of attempts after which the Door Application will turn to Standalone mode the "Turn to Standalone mode after".
  5. Press "Save&Exit" you will be notified that changes will not take affect until application restarts.

This form indicate the application running after the previous manual steps.
