The Clint Role

The basic scenario of entrance is simply student passes his student card through the card reader. The number of the card is read by the client computer using Hooking; that's how simple card readers work.
The client computer checks with its own records whether this student is allowed to enter. If allowed, it opens the door lock for a few preconfigured interval of time; this event is immediately logged. Student validity to enter door are specified by

  • Sending student barcode to Centralized server(Connecting to the doors server Application)
  • Searching barcode in a local prewritten list of barcodes (standalone mode)
  • Sending Query to remote SQL Server where application database is attached into.
