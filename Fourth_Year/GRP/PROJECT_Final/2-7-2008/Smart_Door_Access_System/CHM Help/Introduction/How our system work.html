How our system work? 

Student Enters the Lab
Trigger: A student passes his student card through the card reader. The number of the card is read by the client 
   computer from STDIN as if someone entered the number on the keyboard; that's how simple card readers work. 
   Student with this number is checked to see whether he is allowed to enter door or not this is according to Door 
Application configured mode, as if it runs in stand alone mode then the barcode numbers is searched in the standalone 
file, else if runs under remote SQl Database connection the a query to database is formed finally if it is configured to 
connect to Doors server application then send the number to server via socket and wait for the reply. If positive 
(student allowed), it opens the door lock for a few seconds.

Server Updates List
That is broadcasted periodically after preconfigured time interval, also it is sent to Door Client once its connected to 
the Doors Server, those updates are used by the client if it  runs in standalone alone modes.
