Input / HID - Architecture and Driver Support

Microsoft Windows operating systems support for USB and legacy PS/2-type connections for input devices. USB is the recommended connection for input devices on new PC systems.

Windows also supports uniform access to input devices based on USB Device Class Definition for Human Interface Devices (HID), providing flexible data reporting, typeless data, and arrayed and variable input and output. Windows XP includes HID support for devices connected using legacy input ports. Vendors can extend the functionality of devices by adding filter drivers to the Windows layered driver architecture.

Enhanced Keyboards and Windows. Keyboards with special buttons (to launch a browser, volume controls, and so on) do not need special drivers. In fact, custom drivers add to boot time. Microsoft encourages vendors to implement HID-compatible scan codes and use Windows built-in HID support.
