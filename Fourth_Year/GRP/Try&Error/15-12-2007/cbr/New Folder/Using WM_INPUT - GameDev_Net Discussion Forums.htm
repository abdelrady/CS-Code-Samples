Quote:
MSDN
RIDEV_NOLEGACY
If set, this prevents any devices specified by usUsagePage or usUsage from generating legacy messages. This is only for the mouse and keyboard.
For example, if the mouse TLC is set with RIDEV_NOLEGACY, WM_LBUTTONDOWN and related legacy mouse messages are not generated. Likewise, if the keyboard TLC is set with RIDEV_NOLEGACY, WM_KEYDOWN and related legacy keyboard messages are not generated.
Quote:
MSDN
hwndTarget
Handle to the target device. If NULL, it follows the keyboard focus.

So specifying NOLEGACY and a null hwndTarget should disable all normal input generation for the entire application.
I wonder how you were able to control the window at all..

 rid.usUsage = 0x02;

LRESULT CALLBACK WindowProc(HWND hwnd, UINT msg, WPARAM wParam, LPARAM lParam) {

 }
